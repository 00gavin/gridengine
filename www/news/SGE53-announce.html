Grid Engine 5.3 is now ready.

Regensburg, Germany -- April 3, 2002


The developers of the Grid Engine project finished their work for the Grid Engine 5.3 release.

This release is the first final version of the Grid Engine Open Source project. This release is fully compatible with Sun Grid Engine 5.3 and Sun Grid Engine, Enterprise Edition 5.3 (both products will be available from Sun Microsystems). One Alpha release and two Beta releases preceded this
version. Ready-to-Install courtesy binaries for various Unix platforms including documentation and architecture independent files are available for download.

The software can be installed as "Grid Engine" and "Grid Engine, Enterprise Edition".

The documentation in PDF format has been updated and is provided in the documentation package in the courtesy binary download area of the Grid Engine project site. There are now three manuals available:

Grid Engine Administrator and User Guide
Grid Engine, Enterprise Edition Administrator and User Guide
Reference Manual (for both products)
A couple of bugs and issues of Grid Engine 5.3beta2 have been fixed in this release, among them are:
  • Fixed bug with deletion of non existing array jobs tasks
  • IGNORE_FQDN=FALSE setting did not work when hostnames with domain part where resolved
  • Fixed commd infinite loop when hostname could not be resolved anymore
  • Execution host needs not to be submit or admin host to register at qmaster. Needs to be exec host only
  • Qmon: In share tree now leaf nodes can be added below a project. Added "Clearusage" button. Fixed "Why" and "Interactive" crash. Sort projects alphabetically. Sort "global" host and configuration at top of selection list. Shell_start_mode fix. Fixed share tree "Find" display problem. Projects and user are sorted alphabetically in share tree
  • Fixed deletion of "finished jobs" in spool directory
  • Various security framework installation fixes
  • Added HOWTO for security installation
  • Use less memory and decrease spool file size for tightly integrated parallel jobs
  • Qselect/qstat -U was partially broken
  • Improved messages when commd or qmaster is unavailable
  • Closing socket when "READ ERROR" is encountered (daemons now quickly can reconnect)
  • Qhost output is better formatted
  • Fixed broken qlogin for Solaris
  • Load reports from execd's no longer have lower priority than other messages - this improves the behavior of qmaster in a huge cluster
  • Qacct output is no longer truncated. Better formatting of qacct output
  • Gethostbyname -aname was broken
  • Fixed "missing node XXXXX in dst trees" logging problem
  • The manual pages have been revised


We would like to thank all project members and users who helped with their feedback and contributions  to improve and release Grid Engine 5.3.
